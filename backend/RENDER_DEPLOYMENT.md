# PharmaGuard Backend - Render Deployment Guide

## Prerequisites
- GitHub account with repository pushed
- Render account (sign up at https://render.com)
- MongoDB Atlas account (for database)
- OpenAI API Key
- Groq API Key

## Deployment Steps

### 1. Prepare Environment Variables

You'll need to set the following environment variables in Render:

| Variable | Description | Example/Notes |
|----------|-------------|---------------|
| `JWT_SECRET_KEY` | Secret key for JWT tokens | Auto-generated by Render (recommended) |
| `OPENAI_API_KEY` | OpenAI API key for LLM explanations | Get from https://platform.openai.com |
| `GROQ_API_KEY` | Groq API key for assistant chat | Get from https://console.groq.com |
| `MONGODB_URI` | MongoDB connection string | From MongoDB Atlas |
| `FRONTEND_URL` | Frontend URL for CORS | Leave empty for dev, set to `https://your-frontend.vercel.app` for production |
| `FLASK_ENV` | Environment mode | `production` (Render sets this automatically) |
| `PORT` | Server port | Auto-set by Render |

### 2. MongoDB Atlas Setup

1. Go to https://www.mongodb.com/cloud/atlas
2. Create a free cluster
3. Create a database user with password
4. Whitelist IP: `0.0.0.0/0` (allow all IPs for Render)
5. Get connection string: `mongodb+srv://<username>:<password>@cluster.xxxxx.mongodb.net/pharmaguard?retryWrites=true&w=majority`

### 3. Deploy to Render

#### Option A: Using Blueprint (render.yaml) - Recommended

1. **Push to GitHub**: Ensure your code is in a GitHub repository
2. **Go to Render Dashboard**: https://dashboard.render.com
3. **Create New Web Service**:
   - Click "New +" → "Web Service"
   - Connect your GitHub repository
   - Render will detect `render.yaml` automatically
   
4. **Configure Environment Variables**:
   - Go to "Environment" tab
   - Add all required environment variables listed above
   - Click "Save Changes"

5. **Deploy**:
   - Render will automatically build and deploy
   - Wait for deployment to complete (5-10 minutes)
   - Your API will be available at: `https://pharmaguard-api.onrender.com`

#### Option B: Manual Setup

1. **Create Web Service**:
   - Click "New +" → "Web Service"
   - Connect repository
   - Name: `pharmaguard-api`
   - Region: `Oregon (US West)`
   - Branch: `main`
   - Root Directory: `backend`
   - Runtime: `Python 3`
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `gunicorn app:app`

2. **Set Environment Variables** (as listed above)

3. **Deploy**: Click "Create Web Service"

### 4. Verify Deployment

Test your API endpoints:

```bash
# Health check
curl https://your-api-url.onrender.com/health

# Expected response:
{
  "status": "healthy",
  "service": "Pharmacogenomics API",
  "version": "1.0.0"
}
```

### 5. Update Frontend Configuration

Update your frontend to use the Render API URL:

**In frontend code** (e.g., `UploadPage.jsx`, `SignupPage.jsx`, etc.):

Replace:
```javascript
const response = await fetch('http://localhost:5000/analyze', {
```

With:
```javascript
const API_URL = 'https://your-api-url.onrender.com'
const response = await fetch(`${API_URL}/analyze`, {
```

Or create an environment variable:
```bash
# .env.production
VITE_API_URL=https://your-api-url.onrender.com
```

Then use:
```javascript
const API_URL = import.meta.env.VITE_API_URL || 'http://localhost:5000'
```

### 6. Monitor Your Deployment

- **Logs**: View real-time logs in Render dashboard
- **Metrics**: Monitor CPU, memory usage
- **Health Checks**: Render automatically checks `/health` endpoint
- **Auto-deploy**: Render redeploys on git push to main branch

## Important Notes

### Free Tier Limitations
- **Spin Down**: Free services spin down after 15 minutes of inactivity
- **Spin Up**: First request after spin down takes 30-60 seconds
- **Monthly Hours**: 750 hours/month free
- **Solution**: Upgrade to paid plan ($7/month) for always-on service

### Production Best Practices

1. **CORS Configuration**: 
   - Update `FRONTEND_URL` to your actual frontend URL
   - Don't use `*` in production for security

2. **Secret Keys**:
   - Never commit `.env` file to git
   - Use Render's auto-generated secrets when possible

3. **Database**:
   - MongoDB Atlas free tier includes 512MB storage
   - Set up database backups

4. **Monitoring**:
   - Set up email alerts in Render
   - Monitor API usage and errors

## Troubleshooting

### Service Won't Start
- Check logs in Render dashboard
- Verify all environment variables are set
- Ensure `requirements.txt` includes all dependencies

### CORS Errors
- Verify `FRONTEND_URL` is set correctly
- Check frontend is using correct API URL
- Ensure flask-cors is installed

### Database Connection Fails
- Check MongoDB Atlas whitelist includes `0.0.0.0/0`
- Verify connection string is correct
- Check database user has read/write permissions

### Import Errors
- Ensure all Python dependencies are in `requirements.txt`
- Check Python version compatibility (3.11 recommended)
